## 🎯 Цель проекта
• **Какую задачу решает приложение:** Развитие внимательности, памяти и скорости реакции через игровую механику поиска последовательностей.
• **Почему эта задача важна:** Когнитивные тренировки помогают поддерживать мозг в тонусе и улучшают концентрацию.
• **Целевая аудитория:** Казуальные игроки всех возрастов, люди, желающие улучшить фокус.

---

## 📌 Требования к приложению
**Функциональные требования**
• Генерация последовательности случайных чисел.
• Динамическое размещение элементов на экране без перекрытий.
• Проверка правильности нажатия (по порядку).
• Прогрессия сложности (увеличение количества чисел с каждым уровнем).
• Обработка ошибок (перезапуск уровня при неверном клике).

**Нефункциональные требования**
• **Производительность:** Плавные анимации появления и исчезновения.
• **Удобство использования:** Интуитивно понятный интерфейс без лишних деталей.
• **Надёжность:** Стабильная работа при изменении размеров экрана и системных отступов.

---

## 🧠 Общая архитектура
• **Выбранный подход:** Simple Activity Pattern (близко к MVC, где Activity объединяет логику и отображение).
• **Почему выбран именно он:** Проект имеет небольшой масштаб и единственную игровую сцену, что делает использование сложных паттернов (MVVM/MVI) избыточным.
• **Основные компоненты системы:**
  – `MainActivity`: управление игровым циклом.
  – `RelativeLayout`: динамический контейнер для игровых элементов.
  – `TextView`: визуальное представление чисел.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:** Главный экран и контроллер игры.
**Ответственность:**
- Инициализация UI и обработка системных отступов (Edge-to-Edge).
- Управление уровнями и состоянием игры.
- Создание и размещение игровых элементов.
**Основные методы:**
- `onCreate()` — настройка экрана и запуск первого уровня.
- `startLevel()` — подготовка данных для нового уровня, очистка экрана.
- `createNumberView()` — программное создание "пузырьков" с числами и расчет их случайных координат.
- `onNumberClick()` — проверка последовательности и запуск анимаций.
- `flashErrorAndRestart()` — визуальная индикация ошибки и сброс прогресса уровня.

---

## 🔄 Диаграмма работы приложения
1. Запуск приложения -> `onCreate`.
2. Генерация стартового числа `k` и количества элементов `n`.
3. Размещение `n` чисел на экране в случайных позициях.
4. Ожидание клика:
   - Если кликнули на правильное (следующее) число -> анимация исчезновения -> проверка завершения уровня.
   - Если кликнули неверно -> подсветка красным -> перезапуск уровня через 500мс.
5. При достижении `n > 50` -> Победа.

## 🎨 UI/UX анализ
• **Почему интерфейс сделан именно так:** Максимальный фокус на игровых элементах. Центрированный заголовок уровня и свободное пространство для чисел.
• **Какие принципы использованы:**
  – **Простота:** Отсутствие отвлекающих меню.
  – **Логичность:** Числа визуально выделены как интерактивные объекты.
  – **Доступность:** Крупный шрифт (24sp) и контрастные цвета.
• **Что можно улучшить:** Добавить темную тему, звуковые эффекты и таблицу рекордов.

---

## ⚙️ Работа с потоками
• **Используются:**
  – `Handler`: для создания временной задержки перед перезапуском уровня после ошибки.
  – `Main Thread`: все расчеты координат и UI-операции выполняются в основном потоке из-за их легковесности.
• **Почему выбран этот способ:** Простота реализации для задач с задержкой в UI.
• **Предотвращение зависаний/утечек:** Компактные методы, отсутствие длительных блокирующих операций.

---

## 💾 Работа с данными
• **Где хранятся данные:** В оперативной памяти (переменные класса `MainActivity`).
• **Почему выбран этот способ:** Игровое состояние не требует сохранения между сессиями (на текущем этапе).

## 🐞 Обработка ошибок
• **Логические ошибки:** Обрабатываются через метод `flashErrorAndRestart()`.
• **Runtime ошибки:** Использование проверок на `null` и валидация размеров layout перед расчетом координат.

---

## ⚡ Производительность
• **Оптимизации:** Использование `AlphaAnimation` для легковесных эффектов.
• **Узкие места:** При очень большом количестве элементов (более 100) расчет случайных позиций может потребовать оптимизации алгоритма поиска свободного места.
