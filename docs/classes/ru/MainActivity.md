# Класс: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** `Activity` (Активность)
*   **Назначение:** Это главный контроллер приложения. Он управляет всей игровой логикой, обновлениями интерфейса, переходами между уровнями и взаимодействием с пользователем. Отвечает за генерацию игровой последовательности и механику "Tap Chain" (нажми цепочку).
*   **Взаимодействие:** Взаимодействует с системой Android (жизненный цикл и отрисовка UI), `RelativeLayout` (как контейнер) и динамически созданными объектами `TextView` (игровые элементы).

---

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Использование в логике |
| :--- | :--- | :--- | :--- |
| `level` | `int` | Текущий уровень игры. | Для отображения номера уровня и увеличения сложности. |
| `n` | `int` | Количество элементов для нажатия на текущем уровне. | Определяет, сколько чисел будет создано и размещено на экране. |
| `k` | `int` | Стартовое число последовательности. | Базовое значение для списка чисел на каждом уровне. |
| `nextNumber` | `int` | Число, которое игрок должен нажать следующим. | Используется для проверки правильности нажатия. |
| `numbers` | `ArrayList<Integer>` | Список отображаемых чисел. | Хранит последовательность чисел, которая затем перемешивается. |
| `numberViews` | `ArrayList<TextView>` | Список визуальных элементов на экране. | Хранит ссылки на объекты `TextView` для управления их видимостью или удалением. |
| `mainLayout` | `RelativeLayout` | Главный UI контейнер. | "Холст", на который добавляются или с которого удаляются пузырьки с числами. |
| `levelTextView` | `TextView` | Надпись с текущим уровнем. | Обновляет текст на экране: "Level: X". |
| `random` | `Random` | Генератор случайных чисел. | Для генерации случайных стартовых чисел (`k`) и позиций на экране. |
| `handler` | `Handler` | Инструмент для планирования задач. | Используется для задержки перезапуска уровня после ошибки. |

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Контейнер с данными, если Activity пересоздается. |
*   **Что делает:**
    1.  Включает режим "Edge-to-Edge" (приложение занимает весь экран).
    2.  Устанавливает макет из `activity_main.xml`.
    3.  Инициализирует `mainLayout` и `levelTextView`, находя их в XML.
    4.  Устанавливает слушатель системных отступов, чтобы игра началась только после того, как размеры экрана станут известны.
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Важно:** Это точка входа. Мы используем `v.post(this::startLevel)`, чтобы гарантировать, что макет рассчитан до того, как мы начнем размещать пузырьки.

### Имя метода: `startLevel`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет.
*   **Что делает:**
    1.  Проверяет условие победы (`n > 50`). Если да — показывает сообщение "You Win!" и закрывает приложение.
    2.  Генерирует случайное стартовое число `k` (от 1 до 100).
    3.  Устанавливает `nextNumber` равным `k`.
    4.  Обновляет текст уровня.
    5.  Удаляет все старые представления чисел с экрана.
    6.  Заполняет список `numbers` числами от `k` до `k + n - 1` и перемешивает их.
    7.  Вызывает `createNumberView` для каждого числа в списке.
*   **Когда вызывается:** Вручную при запуске, после прохождения уровня или после ошибки.
*   **Важно:** Перемешивание чисел гарантирует, что последовательность нельзя предсказать по их положению.

### Имя метода: `createNumberView`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `number` | `int` | Конкретное число для отображения в этом пузырьке. |
*   **Что делает:**
    1.  Создает новый объект `TextView`.
    2.  Устанавливает текст, размер, цвет и фон (круг).
    3.  Прикрепляет `OnClickListener` для обработки нажатий.
    4.  Рассчитывает случайную позицию X и Y в пределах экрана.
    5.  Добавляет элемент в `mainLayout`.
    6.  Запускает анимацию "Fade In" (плавное появление).
*   **Когда вызывается:** Внутри цикла в `startLevel` для каждого числа последовательности.
*   **Важно:** Максимальные значения X и Y рассчитываются осторожно, чтобы пузырьки не уходили за края экрана.

### Имя метода: `onNumberClick`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `number` | `int` | Значение числа, на которое нажали. |
    | `numberView` | `TextView` | Визуальный элемент, на который нажали. |
*   **Что делает:**
    1.  Проверяет, равно ли нажатое число `nextNumber`.
    2.  Если правильно:
        - Делает пузырек неактивным для кликов.
        - Увеличивает `nextNumber`.
        - Запускает анимацию исчезновения ("Fade Out").
        - По окончании анимации скрывает элемент.
        - Если это было последнее число, увеличивает уровень и начинает следующий.
    3.  Если неправильно:
        - Вызывает `flashErrorAndRestart()`.
*   **Когда вызывается:** При каждом нажатии пользователя на пузырек с числом.

### Имя метода: `flashErrorAndRestart`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет.
*   **Что делает:**
    1.  Блокирует нажатия на все пузырьки.
    2.  Окрашивает все видимые пузырьки в красный цвет.
    3.  Использует `handler`, чтобы подождать 500 миллисекунд, а затем вызывает `startLevel()` для сброса.
*   **Когда вызывается:** Когда пользователь нажимает неверное число в последовательности.

## 4. Жизненный цикл

*   **onCreate():** Вызывается при создании Activity. Настраивает UI и подготавливает начальное состояние.
*   **Другие методы:** Класс не переопределяет другие методы (например, `onPause`), используя поведение по умолчанию.

---

## 5. Взаимодействие с интерфейсом (UI)

*   **Элементы:** `RelativeLayout` (id: `main`) и `TextView` (id: `levelTextView`).
*   **Привязка:** Использует `findViewById(R.id.name)` для связи XML-элементов с Java-переменными.
*   **События:** Обрабатывает `setOnClickListener` для каждого динамически созданного пузырька.

---

## 6. Взаимодействие с другими компонентами

*   **Intent / Переходы:** Другие Activity не используются.
*   **Данные:** База данных или API не используются; все данные хранятся в оперативной памяти.

---

## 7. Общая логика класса

Класс работает как игровой цикл. Он начинается с генерации набора чисел, которые размещаются случайным образом на экране. Пользователь должен найти наименьшее число и нажать на него, затем на следующее и так далее. Если допущена ошибка, экран "краснеет", и уровень начинается заново. С каждым уровнем количество пузырьков растет, усложняя игру.

---

## 8. Упрощенное объяснение

**Объяснение простыми словами:**
Представь стол, заваленный карточками с числами от 1 до 5 в случайном порядке. Твоя задача — собрать их по очереди: сначала 1, потом 2, потом 3... Если ты возьмешь 3 раньше, чем 2, учитель скажет "Неверно!" и разложит все карточки заново на новые места. Каждый раз, когда ты справляешься, учитель добавляет еще одну карточку (сначала до 6, потом до 7), чтобы стало сложнее. Этот `MainActivity` — как тот самый учитель: он готовит карточки, проверяет твой выбор и организует следующий раунд.

## 🛠 Возможные улучшения
*   **Баг / Плохая практика:** При повороте телефона игра сбрасывается на 1-й уровень. Было бы лучше сохранять состояние в `onSaveInstanceState`.
*   **Улучшение:** Расчет координат не предотвращает наложение пузырьков друг на друга. Стоило бы проверять занятое место перед размещением нового пузырька.
